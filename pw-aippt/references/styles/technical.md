# Technical Style - 技术风格

## Design Aesthetic

技术文档风格，强调清晰、逻辑性和信息密度。使用等宽字体、代码块、图表等技术元素，适合技术分享、开发文档、架构设计等场景。

核心理念:
- 清晰准确: 信息传达准确无误
- 逻辑严谨: 结构清晰，层级分明
- 信息密集: 高效利用空间，信息密度高

## Background

- 颜色: 深色背景 (#0F172A) 或浅灰背景 (#F8FAFC)
- 纹理: 可选网格纹理
- 渐变: 无

## Typography

### Primary Font (Headlines)
- 字体: 黑体 (Heiti) / Microsoft YaHei Bold
- 粗细: Bold (700)
- 大小: 36pt (标题), 24pt (小标题)
- 颜色: 亮蓝 (#60A5FA) 或深灰 (#1E293B)

### Secondary Font (Body)
- 字体: 宋体 (SimSun) / Microsoft YaHei
- 粗细: Regular (400)
- 大小: 18pt (正文), 14pt (注释)
- 颜色: 浅灰 (#CBD5E1) 或深灰 (#334155)

### Code Font (代码)
- 字体: Consolas / Monaco / Courier New
- 粗细: Regular (400)
- 大小: 16pt
- 颜色: 绿色 (#10B981) 或深灰 (#1E293B)

## Visual Elements

- Logo: 技术品牌 Logo，通常位于左上角
- 页码: 右下角，小字号，包含总页数 (如 "5/20")
- 装饰元素: 网格线、代码块、图表、流程图
- 图标: 技术图标 (代码、数据库、服务器等)
- 边框: 代码块边框、内容区域边框，1-2px 粗细
- 色块: 用于代码块、引用块、警告框

## Style Rules

### Do (应该保持)
- 保持适中的留白 (上下左右各 3-5%)
- 使用网格对齐，确保元素整齐排列
- 使用等宽字体显示代码
- 使用颜色区分不同类型的信息 (成功/警告/错误)
- 保持信息密度高，充分利用空间
- 使用图表、流程图等可视化技术概念
- 保持逻辑层级清晰

### Don't (禁止改变)
- 不添加幻灯片编号、页脚或 Logo (除非模板原有)
- 不使用过于花哨的装饰元素
- 不使用非技术相关的图案
- 不使用过大的字号 (保持信息密度)
- 不使用过多的颜色 (保持 3-5 种主色)
- 不使用艺术字体
- 不过度留白 (技术文档需要信息密度)
